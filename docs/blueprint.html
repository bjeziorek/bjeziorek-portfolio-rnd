Cognitive Systems Blueprint

Croatian Learning System — Cognitive Systems Blueprint

Cognitive Landscape

The user operates with fragmented childhood exposure to Croatian, resulting in uneven linguistic structures stored implicitly but not consciously accessible. Motivation is intrinsic but unstable, collapsing when cognitive load rises or when direction is unclear. The user’s after work cognitive state is characterised by fatigue, low activation capacity, and a preference for passive, low effort behaviours.

Key cognitive factors:

  • intrinsic motivation that fluctuates under uncertainty
  • rapid loss of direction without visible structure
  • difficulty perceiving progress in non linear domains
  • high sensitivity to cognitive load
  • strong interest based engagement when tasks feel personally meaningful
  • spontaneous desire to express specific thoughts (“I want to say X in Croatian”)

These patterns indicate a system where motivation, direction, and achievability must be externally scaffolded through structure, feedback, and personalised relevance.

System Landscape

The environment provides zero natural exposure to Croatian. The user’s schedule is irregular, with limited time and low tolerance for high activation tasks. Standard language learning systems assume linear progression and consistent exposure, making them incompatible with the user’s profile.

System pressures:

  • no immersion
  • no natural reinforcement
  • uneven prior knowledge
  • irregular schedule
  • competing low effort behaviours
  • lack of tools matching the user’s unique profile

Design implications:

  • the system must be asynchronous, low threshold, and self generating
  • synthetic data must replace natural exposure
  • progress must be visible even when learning paths are branching
  • the system must adapt to the user’s cognitive state, not the other way around

Friction Map

Activation Friction

The user cannot reliably initiate high effort tasks after work. Low threshold micro actions are required to maintain continuity.

Direction Friction

Uneven knowledge makes linear curricula unusable. The user needs a map that accommodates non linearity without losing direction.

Progress Friction

Non linear gaps make progress invisible. A tracking system must show advancement even when the path is branching.

Meaning Friction

Tasks must feel personally relevant. Interest based content is essential for sustaining engagement.

Opportunity Points:

  • thinking in Croatian during daily activities
  • capturing missing vocabulary in real time
  • integrating Croatian into hobbies (recipes, reading, etc.)
  • using spontaneous linguistic desires as goal seeds

Framework Synthesis

MAKO — Motivational Architecture

  • Motivation: anchored in personal interests
  • Activation: micro actions reduce friction
  • Control: user chooses domains and materials
  • Ownership: goals emerge from the user’s own cognition

MAKO stabilises the motivational system.

MAZE — Non Linear Goal Mapping

The user’s goals are unpredictable and domain based, not linear. MAZE provides:

  • branching structure
  • zoomable layers
  • multi narrative paths
  • direction without linearity

MAZE resolves the contradiction between the need for linear progress perception and the reality of non linear learning.

SPINE — Research Backbone

  • system grounding
  • cognitive modelling
  • behavioural friction mapping
  • hypothesis architecture
  • mixed methods integration
  • systemic implementation

SPINE structures the entire intervention.

CIR + FEDRI — Cognitive Modelling

These frameworks model:

  • regulatory stability
  • behavioural patterns
  • cognitive load dynamics
  • motivational fluctuations

They inform how the system should adapt to the user.

Intervention Blueprint

1. Motivational Lever Mapping

Identify:

  • interests
  • domains of curiosity
  • linguistic desires
  • recurring thoughts the user wants to express

These become the seeds for goal generation.

2. MAZE Narrative Calendar

Insert motivational levers into a branching, zoomable calendar. The researcher observes early usage and refines rules until stable.

3. Synthetic Data Pipeline

The user:

  • attempts to translate interest based texts
  • stops after ~20 gaps
  • records missing vocabulary in native language

AI:

  • extracts vocabulary
  • identifies grammar gaps
  • explains collocations
  • generates personalised practice sentences

This pipeline replaces natural exposure.

4. Practice Loop

  • AI generates Polish → Croatian sentences
  • user translates
  • AI checks
  • progress is logged in MAZE

5. Weekly Iteration

Tools, prompts, and workflows are refined weekly based on:

  • behavioural data
  • cognitive patterns
  • friction points

6. One Month Evaluation

Re run diagnostic materials to measure:

  • linguistic improvement
  • motivational stability
  • system effectiveness

Evaluation Blueprint

Cognitive Metrics

  • stability of motivation
  • sense of direction
  • sense of progress
  • perceived achievability
  • cognitive load patterns

Behavioural Metrics

  • activation frequency
  • micro action completion
  • calendar usage patterns

Systemic Metrics

  • quality of synthetic data
  • alignment between user interests and generated materials
  • effectiveness of MAZE structure

Qualitative Metrics

  • journal entries
  • calendar narratives
  • user reflections

Quantitative Metrics

  • AI graded tasks
  • vocabulary gap reduction
  • frequency of practice loops

Outcome Model

  • goals emerge naturally from the user’s cognition
  • motivation stabilises through ownership and control
  • progress becomes visible through MAZE
  • synthetic data provides continuous exposure
  • micro actions maintain continuity
  • the user reaches functional C1 proficiency through a personalised, adaptive, self sustaining system
