Croatian Learning System — R&D Architecture (Refined Document)

1. Problem Definition

The client has partial, uneven knowledge of Croatian acquired in early childhood, with no exposure in adulthood. Standard language learning tools do not match her profile, and formal proficiency tests cannot accurately place her level. The goal is to design a system that enables her to reach functional C1 proficiency through personalised, motivation driven, synthetic data supported learning.

Context

  • Learned Croatian informally as a child; knowledge is fragmented and inconsistent.
  • Zero current exposure; no natural reinforcement.
  • Adult learner with limited time, irregular schedule, and low tolerance for high activation tasks.
  • Motivation is ambition based and fragile under cognitive load.

Constraints

  • Research team does not know Croatian.
  • No budget for custom tool development.
  • Must rely on frameworks, adaptive workflows, and repurposed tools.
  • Must design a system that becomes self sustaining for the user.

Unknowns

  • Actual current proficiency (cannot be measured with standard tests).
  • What the user wants to be able to say/understand that she currently cannot.
  • Which motivational levers are stable enough to anchor long term progress.

Target Outcome

A personalised, self sustaining learning system that:

  • generates goals from intrinsic motivation,
  • produces synthetic learning materials tailored to the user,
  • tracks progress non linearly,
  • and enables the user to reach C1 level functional competence.

2. System Model

System Map

The learning system consists of:

  • the user (cognitive, motivational, behavioural layers),
  • the environment (zero exposure, irregular schedule),
  • the research framework (SPINE + MAKO + MAZE),
  • the synthetic data pipeline,
  • the tracking and feedback loop.

Actors

  • User: adult learner with uneven prior knowledge.
  • Researcher: designs frameworks, interprets data, refines tools.
  • AI systems: generate synthetic materials, evaluate tasks, provide feedback.

Environment

  • No natural Croatian input.
  • Competing low activation behaviours (TV, scrolling, gaming).
  • Learning is optional, not survival relevant → low priority.

Constraints as Design Inputs

  • Low activation threshold → micro actions required.
  • Irregular schedule → asynchronous, flexible system.
  • No exposure → synthetic data must replace natural input.
  • Uneven knowledge → non linear mapping required.

3. Cognitive Model

Motivational Architecture

The user’s motivation is:

  • intrinsic but unstable,
  • interest driven,
  • sensitive to cognitive load,
  • easily disrupted by lack of direction or visible progress.

Cognitive Constraints

The user quickly loses:

  • motivation,
  • sense of direction,
  • sense of progress,
  • sense of achievability,
  • sense of meaning,
  • belief in eventual success.

Behavioural Patterns

After work, the user defaults to:

  • passive consumption,
  • low effort activities,
  • familiar routines.

Predictive Modelling

Motivation increases when:

  • tasks are interesting,
  • tasks are short and easy,
  • progress is visible,
  • goals feel self generated.

MAKO (Motivation, Activation, Control, Ownership) is the backbone for stabilising these dynamics.

4. Behavioural Friction Map

Activation Friction

  • High effort tasks fail after work.
  • Need for low threshold micro actions.

Direction Friction

  • No clear map of what to learn next.
  • Standard curricula do not match uneven knowledge.

Progress Friction

  • Non linear gaps make progress invisible.
  • Need for a tracker that works with non linearity.

Meaning Friction

  • Tasks must connect to personal interests.
  • Synthetic data must feel relevant.

Opportunity Points

  • Thinking in Croatian during daily activities.
  • Using Croatian in hobbies (recipes, reading, etc.).
  • Capturing missing vocabulary in real time.

5. Hypothesis Architecture

Primary Hypothesis

An intrinsic motivation driven framework combined with MAZE will enable the user to generate goals naturally and maintain long term engagement.

Alternative Hypothesis

If intrinsic motivation is insufficient, low threshold micro actions will maintain continuity and generate enough data for further modelling.

Boundary Condition Hypothesis

If MAZE fails (possible due to its experimental nature), the framework will be iterated until a stable configuration emerges or until iteration exceeds project budget.

Assumptions

  • The user can reliably capture missing vocabulary.
  • Synthetic data can approximate natural exposure.
  • Motivation can be stabilised through structure and feedback.

6. Framework Design

Frameworks Used

  • SPINE — overall research backbone.
  • MAKO — motivational architecture.
  • MAZE — non linear goal tracking and narrative calendar.
  • CIR + FEDRI — cognitive modelling and agent based simulation.

Why These Frameworks

  • SPINE structures the research cycle.
  • MAKO stabilises motivation.
  • MAZE resolves the contradiction between non linear interests and the need for linear progress.
  • CIR + FEDRI model user cognition and behavioural patterns.

Framework Interaction

  • MAKO identifies motivational levers.
  • MAZE organises them into a branching, zoomable calendar.
  • SPINE guides iteration and refinement.
  • Synthetic data pipeline feeds the system with personalised materials.

7. Implementation Plan

Phase 1 — Motivational Mapping

  • Identify intrinsic motivational levers.
  • Capture interests, domains, and linguistic desires (“I want to say X”).
  • Begin collecting missing vocabulary.

Phase 2 — Branching Calendar (MAZE)

  • Insert motivational levers into MAZE.
  • Researcher observes and corrects early usage.
  • Calendar rules refined iteratively.

Phase 3 — Synthetic Data Generation

  • User attempts translations from memory.
  • Gaps captured (~20 per session).
  • AI generates:
    • translation lists,
    • grammar gap lists,
    • collocation explanations.

Phase 4 — Practice Loop

  • AI generates Polish → Croatian sentences.
  • User translates; AI checks.
  • Progress marked in MAZE.

Phase 5 — Weekly Iteration

  • Researcher evaluates tools, data, and user behaviour.
  • Deploys improved versions weekly.

Phase 6 — One Month Evaluation

  • Re run diagnostic materials.
  • Compare entry vs. exit level.
  • Decide whether:
    • strategy is correct,
    • refinement is needed,
    • or assumptions failed.

8. Evaluation Plan

Quantitative Metrics

  • AI graded task results.
  • Frequency of calendar usage.
  • Number of gaps captured.
  • Number of synthetic tasks completed.

Qualitative Metrics

  • Calendar entries.
  • Methodology journal.
  • Surveys.
  • A/B tests of calendar formats.

Behavioural Metrics

  • Activation patterns.
  • Task completion patterns.
  • Cognitive load indicators.

Environmental Metrics

  • Context of learning sessions.
  • Competing behaviours.

Success Criteria

  • Increased sense of direction.
  • Increased sense of progress.
  • Increased belief in achievability.
  • Stable or rising motivation.
  • Measurable linguistic improvement.

9. Risks & Mitigations

Cognitive Risks

  • Overload → mitigated by micro actions and MAZE.
  • Loss of motivation → mitigated by MAKO.

Behavioural Risks

  • Irregular schedule → asynchronous system.
  • Low activation → ultra low threshold tasks.

Systemic Risks

  • No natural exposure → synthetic data pipeline.
  • Uneven knowledge → non linear mapping.

Operational Risks

  • Researcher does not know Croatian → AI assisted diagnostics and synthetic materials.
