Product Strategy Case Study

Croatian Learning System — Product Strategy Case Study

Challenge

Design a learning system for an adult user with fragmented childhood exposure to Croatian, no current immersion, and an irregular schedule. Standard courses and proficiency tests fail because her knowledge is uneven and non linear. The goal is to help her reach functional C1 proficiency through a personalised, motivation driven approach.

Context

The user:

  • learned Croatian informally as a child,
  • retained partial, inconsistent knowledge,
  • has no exposure in adulthood,
  • works full time with unpredictable hours,
  • loses motivation quickly when tasks feel heavy or directionless.

The research team:

  • does not know Croatian,
  • cannot build custom tools due to budget constraints,
  • must rely on frameworks, adaptive workflows, and AI generated synthetic data.

User Insights

  • Motivation is intrinsic but fragile; it collapses when progress is invisible.
  • After work, cognitive load is high; activation threshold must be extremely low.
  • The user gravitates toward hobbies and passive activities — these can be leveraged as learning entry points.
  • She wants to express specific thoughts (“I want to say X in Croatian”), which can be used as personalised goal seeds.
  • Missing vocabulary appears constantly during daily thinking; capturing it is a powerful data source.

System Insights

  • Standard linear curricula fail because the user’s knowledge is non linear.
  • Progress must be visible even when learning paths are branching and unpredictable.
  • Synthetic data can replace natural exposure if generated from the user’s own interests and gaps.
  • A non linear tracker is required to reconcile spontaneous goals with the need for direction.

Opportunities

  • Use MAKO to stabilise motivation through ownership and control.
  • Use MAZE to create a branching, zoomable narrative calendar that maps non linear goals.
  • Use AI to generate personalised synthetic materials from the user’s captured gaps.
  • Use micro actions to maintain continuity on low energy days.
  • Use weekly iteration to refine tools and frameworks in real time.

Solution Architecture

1. Motivational Mapping

Identify intrinsic motivational levers: interests, domains, linguistic desires, and recurring thoughts the user wants to express.

2. Branching Calendar (MAZE)

Insert motivational levers into a non linear, multi layered calendar that provides direction without forcing linearity.

3. Synthetic Data Pipeline

Generate personalised materials by:

  • capturing missing vocabulary during daily thinking,
  • translating interest based texts until ~20 gaps appear,
  • using AI to extract vocabulary, grammar gaps, and collocations.

4. Practice Loop

AI generates Polish → Croatian sentences; the user translates; AI checks; progress is logged in MAZE.

5. Weekly Iteration

Tools, prompts, and workflows are refined weekly based on behavioural data and cognitive patterns.

6. One Month Evaluation

Re run diagnostic materials to measure progress, validate hypotheses, and adjust strategy.

Prototype / Intervention

The intervention consists of:

  • a MAZE based narrative calendar,
  • a structured learning journal,
  • a synthetic data generation workflow,
  • a personalised prompt set for AI,
  • weekly evaluation cycles,
  • entry and exit diagnostics using synthetic texts.

This creates a self sustaining learning ecosystem driven by the user’s own cognition and interests.

Expected Outcomes

  • Increased sense of direction through MAZE.
  • Increased motivation through MAKO.
  • Visible progress through synthetic data loops.
  • Reduced activation friction through micro actions.
  • A personalised, adaptive path toward C1 proficiency.
  • A stable, long term learning system the user can maintain independently.

What I Learned

  • Non linear learners require non linear tracking systems.
  • Synthetic data can effectively replace natural exposure when grounded in user generated content.
  • Motivation stabilises when users feel ownership over goals and materials.
  • Behavioural friction mapping is essential for designing low activation workflows.
  • Framework driven design (SPINE, MAKO, MAZE) enables high complexity problems to become tractable and structured.
