How I Document My Work

My workflow has three layers, each producing different types of documents. Not every project requires all of them — the system adapts to the complexity of the problem.

1. Raw Models

Every project begins here. This layer has two modes depending on the required resolution.

Raw cognitive dump

(used when FEDRI LOW-RES or MID-RES is sufficient)

This is the unfiltered modelling phase where patterns emerge, contradictions surface, and the architecture of the solution begins to take shape. It’s enough for problems that don’t require identity modelling or multi‑agent simulation (e.g., designing a learning system, motivational workflow, or behavioural pipeline).

Narrative raw modelling essays

(used when FEDRI HI-RES is required and/or when I need to bypass autocensorship)

This technique gives me access to deeper dynamics, emergent behaviour, and high‑resolution cognitive modelling. Writing in a playful, deadpan “scientist mode” slows my thinking to a documentable pace and bypasses self‑censorship, allowing insights to be captured before they disappear. These essays generate behavioural material that later feeds into my forensics work. I sometimes publish them on my blog — usually private projects, but occasionally commercial ones when they contain no NDA‑protected content or can be anonymised. Teams may request these narratives, though they often run 15–20 pages of uninterrupted thought flow.

2. Deep Analysis

(optional, required only for FEDRI HI-RES)

Deep Analysis is used for high‑complexity problems such as designing new architectural modules or analysing emergent behavioural patterns. Over the last decade I’ve built a private agentarium — a library of 100+ agents and 1000+ situational templates — which I recombine into controlled behavioural probes and analytical scenarios. This layer operates entirely in FEDRI HI-RES mode and consists of two complementary processes: behavioural probe and behavioural forensics. Together, they form the high‑resolution analytical pipeline used when a problem requires detailed modelling of dynamics, cognition, and emergent behaviour.

2A. Behavioural Probe (synthetic data generation)

(FEDRI HI-RES — narrative mode)

Behavioural probes are used when no empirical behavioural data is available and synthetic data must be generated manually. In this mode, FEDRI operates narratively, producing high‑resolution micro‑scenes that simulate real‑world dynamics under controlled cognitive conditions. These probes allow me to explore hypotheses, observe emergent behaviour, and map system interactions before any formal analysis. They serve exclusively as input for behavioural forensics and never appear in refined documents.

2B. Behavioural Forensics (analysis)

(FEDRI HI-RES — analytical mode)

Behavioural forensics is the analytical phase applied either to real behavioural data (e.g., recorded interactions, observational material) or to synthetic behavioural probes when empirical data is unavailable. In this mode, FEDRI operates analytically, extracting heuristics, parameters, vectors, and system‑level dynamics from the source material. This process transforms raw behavioural content into structured insights that feed directly into refined documentation, without including any narrative fragments.

Additional notes

Note on terminology: “character” vs. “agent”

In behavioural probes (FEDRI HI‑RES — narrative mode), I use the term character to describe the acting entity inside the micro‑scene. A character is a narrative construct with access to emotional memory, identity dynamics, and tacit cognitive processes. This form is optimised for generating high‑resolution synthetic behavioural material.

In behavioural forensics (FEDRI HI‑RES — analytical mode), I use the term agent. An agent is the analytical representation of the same entity: stripped of narrative form, expressed through parameters, heuristics, vectors, and system‑level dynamics. Agents are explicit, fully logical, and used exclusively for analysis.

Note on narrative and agentarium content

All narrative content remains private; only abstracted outputs are shared externally. Small fragments of narrative material or agentarium content may occasionally appear in the Raw Models layer, but never in refined documents. All refined outputs contain only abstracted, formalised structures — never narrative scenes, micro‑simulations, or behavioural probes.

Any other format required by the team

If a team has its own documentation format, I adapt to it. When needed, I also translate my framework outputs into other APIs — vectors, parameters, or structures ready for code.

Technical Execution

Depending on the project, I also produce technical specifications, code documentation, JSON schemas, architectural diagrams, and full‑stack implementations. The format always adapts to the problem: whatever the system needs, I build.
