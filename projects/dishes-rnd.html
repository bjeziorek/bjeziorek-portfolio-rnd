Dishwashing System Reconstruction

Dishwashing System Reconstruction

SPINE · CIR · FEDRI applied to a domestic behavioural system

1. Problem Definition

Context

Chronic kitchen chaos, recurring dish pile ups, and avoidance loops around washing dishes. The system behaved unpredictably: sometimes dishes were washed immediately, sometimes ignored for days. The user believed this was a “personality flaw” or “quirk of motivation”.

Constraints

  • Limited counter space.
  • One drying rack with mixed, non dedicated items.
  • Heterogeneous dish set (random plates, bowls, mugs).
  • High cognitive load from work → low bandwidth for domestic tasks.
  • Sensory friction (visual clutter, tactile discomfort).
  • No automation (no dishwasher).

Unknowns

  • Why avoidance triggered so strongly.
  • Why the system collapsed despite low task complexity.
  • Why motivation was irrelevant to outcome.
  • Why the user experienced the task as “impossible” despite capability.

Target Outcome

A stable, low friction dishwashing system that operates without motivation, discipline, or executive effort.

2. System Model

System Map

  • Inputs: dirty dishes, cooking events, sensory load, available space.
  • Processes: dish placement, washing, drying, storing.
  • Outputs: clean dishes, clutter, avoidance loops.
  • Feedback loops: clutter → overwhelm → avoidance → more clutter.

Actors

  • Primary agent: user (high cognitive load, ND adjacent processing patterns).
  • Secondary agents: environmental constraints (space, objects, sensory triggers).
  • Emergent agent: “dish pile” acting as a self reinforcing behavioural attractor.

Environment

  • Small kitchen.
  • High visual noise.
  • No clear affordances for dish flow.
  • Drying rack acting as a bottleneck.

Constraints as Design Inputs

  • System must operate under low regulation (CIR 1–2).
  • System must reduce sensory load, not increase it.
  • System must automate behaviour, not require willpower.

3. Cognitive Model

User Cognition

  • High sensitivity to clutter.
  • Strong avoidance when overwhelmed.
  • Executive bandwidth reserved for work, not chores.
  • Automatic micro behaviours dominate domestic tasks.

Motivational Levers

  • Not effective: discipline, reminders, guilt, “just do it”.
  • Effective: reducing friction, clarifying affordances, predictable flow.

Behavioural Patterns

  • Avoidance loop triggered by visual overload.
  • “One dirty dish” → “system collapse” cascade.
  • No stable mapping between dish and purpose.

Cognitive Load Factors

  • Mixed dish types = decision fatigue.
  • Drying rack = ambiguous state → cognitive noise.
  • No dedicated items = no automatic behaviour.

Predictive Modelling

FEDRI predicted that:

  • The system fails not due to motivation but due to structural ambiguity.
  • Increasing the number of dedicated dishes reduces decision load.
  • Removing the drying rack creature eliminates a major friction node.
  • Assigning each dish a purpose stabilises behaviour.

4. Behavioural Friction Map

Activation Thresholds

  • Washing triggered only when clutter < threshold.
  • Threshold exceeded almost instantly.

Failure Points

  • Drying rack as a permanent “limbo state”.
  • No clear start/stop boundaries.
  • Mixed dish types → micro decisions → overwhelm.

Opportunity Points

  • Dedicated dish sets.
  • Clear sensory boundaries.
  • Reduced object variety.

Levers for Change

  • Reduce cognitive load.
  • Reduce sensory noise.
  • Increase environmental clarity.
  • Assign fixed roles to objects.

5. Hypothesis Architecture

Primary Hypothesis

Increasing the number of dedicated dishes and reducing environmental ambiguity will stabilise the dishwashing system.

Alternative Hypotheses

  • Reducing dish count will reduce clutter (rejected by model).
  • Increasing motivation will improve outcomes (rejected).
  • Scheduling dishwashing will stabilise behaviour (rejected).

Boundary Condition Hypotheses

  • System fails if sensory load > threshold.
  • System fails if dish roles are ambiguous.
  • System stabilises only if object behaviour mapping is explicit.

Assumptions

  • User operates under low regulation.
  • System must self regulate.
  • Behaviour must be automatic, not effortful.

6. Framework Design

Chosen Frameworks

  • SPINE — system grounding, prediction, intervention.
  • FEDRI — avoidance loop reconstruction, internal state modelling.
  • CIR — regulation thresholds and behavioural readiness.

Why These Frameworks

  • SPINE maps the architecture.
  • FEDRI exposes hidden mechanics.
  • CIR explains why motivation is irrelevant.

How They Interact

SPINE identifies constraints → FEDRI reconstructs loops → CIR determines feasibility → combined output produces intervention.

7. Implementation Plan

Phases

  • Remove drying rack creature.
  • Empty cupboard.
  • Assign dedicated roles to each dish.
  • Purchase missing vessels to complete the system.
  • Rebuild environment with clear affordances.

Tools

  • SPINE system grounding.
  • FEDRI loop tracing.
  • CIR readiness modelling.

Data Flows

  • Observed behaviour → friction map → model prediction → intervention.

Feedback Loops

  • Clean dishes reinforce system stability.
  • Reduced clutter reduces avoidance.
  • Dedicated items reduce decision load.

Iteration Cycles

  • Observe → adjust dish roles → refine environment.

8. Evaluation Plan

Quantitative Metrics

  • Time to wash a dish.
  • Number of dishes in sink.
  • Frequency of system collapse.

Qualitative Metrics

  • Perceived overwhelm.
  • Emotional load.
  • Sense of control.

Behavioural Metrics

  • Automaticity of washing.
  • Avoidance loop frequency.
  • Regulation level during task.

Environmental Metrics

  • Visual noise.
  • Object ambiguity.
  • Flow clarity.

Success Criteria

  • System operates without motivation.
  • No avoidance loops.
  • Stable behaviour across days.

9. Risks & Mitigations

Cognitive Risks

  • Overwhelm during reorganisation → mitigate by staging.
  • Decision fatigue → mitigate by pre assigning roles.

Behavioural Risks

  • Regression under stress → mitigate by reducing object variety.

Systemic Risks

  • New clutter attractors → mitigate by clear affordances.

Operational Risks

  • Wrong dish assignments → mitigate by iterative refinement.
